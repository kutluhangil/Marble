# MARBLE v1 Implementation Plan

**Goal:** Build a hyper-realistic, slowly rotating Earth on an editorial-light landing page with real-time data layers (quakes, ISS, city weather, optional flights), deployable to Vercel.

**Architecture:** Next.js 14 App Router + TypeScript strict. R3F scene: a spinning world group (Earth + clouds + data points) under a fixed real-time sun, fixed atmosphere/contact-shadow, camera via OrbitControls. Pure logic (geo math, normalization) is React/Three-free and unit-tested. Data flows API route → normalize → poller → Zustand → render.

**Tech Stack:** Next.js 14, React 18, Three.js, @react-three/fiber, @react-three/drei, Zustand, Framer Motion, Tailwind, Vitest, sharp (texture script).

Reference spec: `docs/specs/2026-05-31-marble-v1-design.md`.

- `components/layers/*` — instanced points, ISS marker, orbit path.
- `components/landing/*`, `components/ui/*`, `components/motion/*` — DOM presentation.
- `lib/textures/loader.ts`, `scripts/fetch-textures.mjs` — assets.

Tests: `tests/geo/*.test.ts`, `tests/data/normalize.test.ts`.

```ts
    expect(pts[8].length()).toBeCloseTo(1,4);
  });
});
```
- [ ] Run → FAIL.
- [ ] Implement: convert endpoints to unit vectors (radius 1), slerp by t∈[0,1], scale each by `1 + maxAlt·sin(π·t)` (peak at middle). Use `Vector3.slerp` via quaternion or manual slerp.

- [ ] Commit: `feat: great-circle arc generation`.

### Task 2.3: sun-position (test-first)
**Files:** `lib/geo/sun-position.ts`, `tests/geo/sun-position.test.ts`

- [ ] Failing test: at the 2025 June solstice (~2025-06-21T00:00Z) subsolar latitude ≈ +23.4° (±1°); at equinox (~2025-09-22) ≈ 0° (±1.5°); `sunDirectionWorld(date)` returns a unit-length `Vector3`.
```ts
import { describe, it, expect } from 'vitest';
import { subsolarPoint, sunDirectionWorld } from '@/lib/geo/sun-position';
describe('subsolarPoint', () => {
  it('~+23.4 lat at June solstice', () => {
    expect(subsolarPoint(new Date('2025-06-21T00:00:00Z')).lat).toBeCloseTo(23.4, 0);
  });
  it('~0 lat at September equinox', () => {
    expect(Math.abs(subsolarPoint(new Date('2025-09-22T18:00:00Z')).lat)).toBeLessThan(1.5);
  });
  it('sun direction is unit length', () => {
    expect(sunDirectionWorld(new Date()).length()).toBeCloseTo(1, 4);
  });
});
```
- [ ] Run → FAIL.
- [ ] Implement low-precision NOAA solar position: day-of-year → declination; UTC → subsolar longitude = -15°·(UTC_hours - 12) (approx, ignoring equation of time, within tolerance). `sunDirectionWorld` = `latLngToVector3(subsolar,1).normalize()`.
- [ ] Run → PASS.
- [ ] Commit: `feat: real-time subsolar point and sun direction`.
